Send a POST request to https://api.moxiworks.com/api/transactions

When creating an transaction using the Moxi Works platform API, format your data using the following parameters.

Required Parameters Are In Red

moxi_works_agent_id This is the Moxi Works Platform ID of the Agent which this Transaction object is to be associated with. This will be a string that may take the form of an email address, or a unique identification string. This data is required and must reference a valid Moxi Works Agent ID for your Transaction Create request to be accepted.

moxi_works_contact_id This is the unique identifer for the Contact that this Transaction regards. This will match the Contactuuid attribute.

name A brief, human readable title that will be shown to the agent as the subject of the Transaction that you are creating.

notes Brief, human readable content that will be shown to the agent as notes about the Transaction that you are creating.

address Brief, human readable content that will be shown to the agent as notes about the Transaction that you are creating.

due_at A Unix timestamp representing the point in time when the transaction associated with this Transaction object should be completed by.

duration The timespan (in minutes) that the transaction for this Transaction object is expected to take.

Use either commission_percentage or commission_percentage.

Transaction Create Response Payload

The following attributes make up the payload of the Transaction response for a Create request.

Attribute Type
moxi_works_agent_id String
partner_contact_id String
name String or null
description String or null
due_at Integer
duration Integer or null
status String or null
created_at Integer
completed_at Integer or null

moxi_works_agent_id This is the Moxi Works Platform ID of the Agent. This will be a string that may take the form of an email address, or a unique alphanumeric identification string.

partner_contact_id This is the unique identifer you use in your system that has been associated with the Contact that you are creating this Transaction for.

name A brief, human readable title that will be shown to the agent as the subject of the Transaction.

description Brief, human readable content that will be shown to the agent as the body of the Transaction.

due_at A Unix timestamp representing the point in time when the transaction for this Transaction object should be completed by.

duration The timespan (in minutes) that the transaction for this Transaction object is expected to take.

created_at A Unix timestamp representing the point in time when the Transaction object was created.

completed_at A Unix timestamp representing the point in time when the transaction for this Transaction object was completed. This should be null if the transaction has not yet been completed.
